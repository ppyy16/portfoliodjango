Artistic Groove

This game was a prototype and a part of the Computational Thinking K-12 Project that I did with Professor Rich Nguyen, at UVA. You can read more about the project, in my paper below. In this game the aim is to recreate the image (randomly selected) using various shapes. You can rotate these shapes by right clicking and place them appropriately on the canvas to create an abstract version of the image provided. This game was coded in C#, in Unity.
You can see the code here in my Github Repo

NASA Power Application

The NASA power application redesign, was a UX/UI and partial recoding of the existing web page which can be seen here. This webpage is meant to be a helpful visualization of the location of solar panels and other NASA equipment that may be useful to the public. However, the UI/UX for this page is horrendously awful for the user and I was informed that the software was developed with a commercial software generator, causing the program to have pages upon pages of machine generated code which was hard to manipulate and update which had left the program in an awful condition. My job as a contractor for Booz Allen Hamilton was to redesign the page above and get done as much of a shell of the program as I could before my month long internship came to an end. During my time, I questioned employees about the usability of the program and what features need to be kept, scrapped, or improved and drew wireframes. I then came up with an interface design in Adobe XD as you can see below.

During my time, I also developed a semi-working interface through Javascript, HTML, CSS, and Django which you can see the code of in my GitHub Repo.

Photo Mini-Market!

In this project, me and a group of classmates and friends decided to create image manipulation "filters" by using seam carving and Voronoi diagrams. I will only be speaking about Voronoi as that was my contribution to the project.

Initially, my idea of coding and manipulating this algorithm to work for image modification was to only gather points which have high entropy, meaning that they would capture the significant parts of the image where data retention is neccesary. After making this somewhat work, I realized that the image simply did not look like the original enough and realized that a randomized set of points would do just as well, if not better at preserving the image as well as reducing the error in detecting points of entropy in the first place which resulted in the final program. You can see the results below, turned out pretty well if I may say so myself!
